# ADR-011 — Agent-runtime execution slice: LangGraph compilation, capability execution & real HITL

- **Status:** Accepted (Step 3 vertical slice)
- **Date:** 2026-07-08
- **Related:** `amendia_platform_contracts_v1.md` (contracts 4 dispatch, 5 HITL), `amendia_contracts_reference.md`
  (§2.4 HITL modes, §6 dispatch, §7 HITL semantics), `wire-transfer-exception-reference.md` (§6 bindings, §7 BPMN),
  `amendia_build_plan.md` (Step 3); ADR-007 (stub), ADR-008 (ingestor), ADR-009 (agent-runtime foundation),
  ADR-010 (process-registry); the four services' READMEs.
- **Advances:** the "execution (LangGraph, BPMN compilation, dispatch consumers, HITL resume) is the next slice"
  item in `amendia_project_brief.md`.
- **Superseded in part by ADR-016:** the "simulation seam / minimal real LLM path" (Part D below) is replaced —
  `llm` capabilities now run a real, provider-agnostic model via **polyllm + ConfigForge** (config-driven, per
  capability or platform-wide), and `mcp` falls back to simulation rather than raising `NotImplementedError`.
  Simulation remains available via `AGENTRT_SIMULATION_MODE=true`.

## Context

After ADR-009 (contracts + persistence + seed) and ADR-010 (authoring + validation + `/resolve`), the
platform could *describe and validate* a bank's exception process but not *run* one. The dispatch/HITL
lifecycle was dormant: the ingestor stopped at `received`, and the agent-runtime had empty
`process_instances`/`hitl_tasks` collections and read-only APIs.

This slice makes **one exception flow end-to-end**: generated by the stub → ingested → resolved →
dispatched → executed as a compiled process → gated by **real human approvals via API** → completed,
with everything checkpointed in Mongo. It settles the ADR-009 open question in favour of
**native BPMN interpretation with LangGraph** (no external BPMN engine).

## Decision

### Part 0 — shared BPMN parser + envelope (remove drift before building)

- **`libs/amendia_bpmn`** (new package): the registry's Stage-1 BPMN parser (element subset, `BpmnModel`,
  reachability, gateway checks) was lifted out so the **registry validates and the runtime compiles the
  same subset from one source**. The registry's `validation/bpmn.py` is now a thin adapter that maps the
  shared parser's neutral findings into its `ValidationReport` (identical error codes → registry tests
  unchanged). The shared `Flow` gained the raw `conditionExpression` text the compiler needs.
- **`WireExceptionEnvelope`** moved from the stub into `amendia_contracts.wire_exception` so the stub
  (producer) and the runtime (consumer) validate one shared model; the stub keeps a re-export shim.

### Part A — ingestor: resolve + dispatch (activates the dormant lifecycle)

After recording `received`, the ingestor calls the registry `POST /resolve`:
- **match** → persist the resolution, transition `received → dispatched`, publish `exception_dispatched`
  (contract 4) via a new publisher (routing key `ingestor.exception_dispatched.v1`,
  `trace.correlation_id = exception_id`, `causation_id = <exception_raised event_id>`);
- **404 no-match** → new terminal state `no_process`;
- **registry unreachable** → stay `received`; a periodic **retry sweep** re-resolves stale records.

A second durable consumer binds the runtime's replies (`agent_runtime.dispatch_accepted/rejected.v1`)
and drives `dispatched → accepted/rejected`. All transitions are **guarded** (expected-state filter) so
redelivery/replay is a safe no-op.

### Part B — agent-runtime: dispatch consumer + instance lifecycle

A durable consumer binds `ingestor.exception_dispatched.v1`. The `DispatchService`:
idempotency key `(exception_id, pack_key, pack_version)` → existing instance re-`accepted`;
else **fetch the envelope from `fetch_url`** (2 retries → `dispatch_rejected(fetch_failed)`), validate it
(`envelope_invalid`), **load the pack from the registry API** — manifest + resolution + BPMN + pinned
capability descriptors + pinned artifact schemas — (unknown/inactive → `unknown_pack`/`pack_not_active`),
create the `process_instances` doc (`created`), publish `dispatch_accepted`, and start execution.
The `(manifest, resolution, compiled graph, schemas, descriptors)` **bundle is cached per
(pack_key, pack_version)** — immutable once active.

### Part C — the compiler: BPMN + manifest + resolution → LangGraph `StateGraph`

`app/engine/compiler.py`, deterministic and Mongo-free (unit-testable):
- each bound serviceTask/userTask → one node (the generic task runner); startEvent → entrypoint;
  endEvent → a marker node recording `outcome` → `END`; sequenceFlow → edge.
- **exclusiveGateway** → a conditional edge whose router evaluates flow conditions. Supported expression
  subset: `<dotpath> = "lit"` / `!= "lit"` (both `=`/`==`, whitespace-tolerant) against `state.artifacts`
  (first dot-path segment = artifact name, per `gateway_variables`); anything else is a **compile error
  naming the gateway**. Default flow when nothing matches; no default + no match → a compiled-in failure
  sink that fails the instance.
- **parallelGateway → compile error** (out of scope). The seed BPMN was **linearized** accordingly
  (`ApplyRepair → NotifyParties → RecordResolution → End_Resolved`); see `wire-transfer-exception-reference.md` §8.
- **Checkpointing:** `langgraph-checkpoint-mongodb`, `thread_id = process_instance_id`, dedicated
  runtime-private collections in the same `amendia` db — a checkpoint at every node boundary is the audit trail.

### Part D — generic task runner + capability executor (simulation seam)

The task runner (`app/engine/task_runner.py`) per node: gather inputs → (pre-gate) → execute → validate
outputs against the **pinned** artifact schema (jsonschema 2020-12) → (post-gate) → commit → append
`actor_log`. Retries honour `max_retries` **only if `idempotent`**; a validation failure retries once if
idempotent, else fails. The **executor** (`app/engine/executor/`) dispatches by capability kind — `skill`
imports `runtime.entrypoint`; `llm`/`mcp` route to a paired **simulation** skill when
`AGENTRT_SIMULATION_MODE=true` (default), with a minimal real LLM path behind the flag and MCP raising
`NotImplementedError`. The 10 simulated capabilities (`app/capabilities/wire_repair/`) are deterministic
and envelope-aware; the five skill descriptors' entrypoints now point at them.
> **Superseded by ADR-016:** the "minimal real LLM path" is now the real, config-driven polyllm + ConfigForge
> path (`llm` → real model; `mcp` → simulation fallback, no longer raising).

### Part E — HITL: interrupt, task, decision API, resume

Human gates use LangGraph `interrupt(payload)`/`Command(resume=…)`. **Nodes stay synchronous and pure**
(run via `asyncio.to_thread` — the Mongo saver is sync); the async **engine** materializes the `HitlTask`
doc from the interrupt payload — mode-derived `allowed_decisions` (§7 table), pinned-schema payload
snapshots, `proposed_actions` for `approve_actions`, and **`sod.excluded_users` computed from the pack's
`distinct_actor` policies × the instance's `actor_log`** — sets the instance `waiting_hitl`, and publishes
`hitl_task_created`. The decision API:
- `POST /hitl-tasks/{id}/claim {user_id, role}` — 409 unless `open`, 403 if SoD-excluded, role-claim stub.
- `POST /hitl-tasks/{id}/decide {user_id, decision, comment?, edits?, approved_action_ids?}` — validates
  claim + `decision ∈ allowed_decisions`, re-checks SoD, re-validates `edit_and_approve` edits against the
  pinned schema (400 on failure), persists the decision, publishes `hitl_task_decided`, and **resumes the
  graph on the thread** (`waiting_hitl → running → next gate | terminal`).

Because LangGraph replays **only the interrupted node** on resume, deterministic simulation capabilities
are safe to re-run (propose-mode has no side effects; execute-mode runs once, post-approval).
**Crash safety:** `waiting_hitl` needs no recovery; a startup sweep re-invokes `running` instances from
their last checkpoint.

### Part F — completion, status & observability

Graph `END` → instance `completed` + `outcome`; unhandled node error → `failed` + detail; both publish
thin `process_completed`/`process_failed` events. `GET /instances/{id}` returns status + artifact names +
`actor_log` + HITL-task links; `GET /instances/{id}/state` (flag-guarded) returns the checkpointed state.

## Consequences

- The full path runs on the live stack: `tools/demo_wire_repair.sh` drives an AC01 exception to a
  `completed` instance through **all four HITL modes** (review_after, manual, approve_result,
  approve_actions), with **SoD** blocking the drafting analyst from approving their own repair;
  checkpoints for every node boundary exist in Mongo; the BE04 path parks at the `ObtainInfo` manual task.
- **Native-interpret settled:** LangGraph interprets the validated subset; the BPMN XML stays
  annotation-free (execution metadata in the manifest), closing the ADR-009 engine-choice question.
- **Architectural seam:** graph nodes are pure/sync and do no IO — all Mongo/Rabbit/registry work is in the
  async engine, and human gates are re-entrant (interrupt → materialize task → resume), so the same code
  powers both fresh runs and crash recovery.
- **Deliberately deferred:** real MCP/LLM beyond the flagged minimal path, timers/escalation/expiry,
  parallel gateways, compensation, authn/z beyond the role-claim stub, and notification-service
  consumption of the published events (events are emitted; consuming them is the platform service's job).
- **Two non-obvious traps recorded for maintainers:** (1) `langgraph-checkpoint-mongodb` ships only a
  **sync** `MongoDBSaver`, hence the `to_thread` execution model; (2) `mongomock-motor` mishandles
  `find_one_and_update(projection=…, return_document=AFTER)` (returns `None` on a real match), so the
  guarded-transition repos drop the projection kwarg and pop `_id` manually. Both are transparent on real
  Mongo. Editing the seed requires `docker compose … down -v` (active packs are immutable).
